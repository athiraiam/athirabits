Introduction to software testing


Software testing

Software testing is the process of verifying the functionality of an application against the requirement to identify or catch defects in the software. Software testing is categorized into three software testing method:

Black box testing: Testing the application against the requirement without looking into the internal code, structure and design. Here, tester look into the requirement then test each and every component thoroughly against the application. Black box testing is also known as closed box testing, functionality testing and behavioural testing.
White box testing: Testing line of code, internal structure , design and implementation by looking into the source code of the software. Here, we test each and every line of code and ensure that program is working as expected. White box testing is also known as glass box testing, transparent box testing, open box testing, unit testing, structural testing and mutation testing.
Grey box testing: It is combination of white box testing and black box testing. Here, testers have limited knowledge on coding as well as internal structure and have knowledge on understanding behaviour of application under test.

Types of software testing

Manual testing: Testing the application again and again manually against the requirement to find the defects is called as manual testing.
Automation testing: Test engineer will write a program or script and run it against the application automatically to check the result as pass or fail. This is called automation testing.

Importance of software testing

  • While building a software people could make error or mistake which will lead to failure of software, loss of life, money, time, business, reputation etc. Testing ensures that error or defects are found in the early stages, there by making the software or product more reliable and secure.
  • Well tested product guarantees or ensures the quality of the product.
  • If the defect is found in the earlier stage that is before releasing it to end user then the cost of fixing defects are less. Cost of findng defects and fixing defects increases over time. Hence it is recommended a Thorough testing before releasing any product to end user.
  • By testing we gain confidence on the software there by giving satisfaction to their customers. It ensures the best user experience.

    • Disadvantages of manual testing

  • Manual testing is time consuming
  • Manual testing is less reliable as it is conducted by humans and we might end up making unintentional mistake.
  • Expensive over automation
  • It is tough job as we have to test application or software again and again manually.
  • Since we are manually testing it, so the time taken to give the solution to the customer will take time i.e turn around time is more.
